Guía de Carga y Despliegue en Git

1. Configuración de Subida de Datos (Opción A)

Has seleccionado subir la información directamente en el repositorio Git. Esta opción permite que la web cargue automáticamente el archivo al abrirse.

Asegúrate de colocar el archivo reporte UE002.csv directamente en la raíz de este proyecto. La estructura del repositorio debe verse así:

PENDIENTES_UE002/ ├── lib/ │ └── papaparse.min.js ├── index.html ├── style.css ├── app.js ├── README.md └── reporte UE002.csv <-- Colocar aquí

2. Subida a un Repositorio Git (GitHub/GitLab)

Abre una consola en la carpeta de este proyecto y ejecuta los siguientes comandos para subirlo a tu repositorio remoto:

git init git add . git commit -m "Initial commit: SISGED Document Tracker web application with dataset" git branch -M main git remote add origin https://github.com/TU_USUARIO/TU_REPOSITORIO.git git push -u origin main

3. Publicación Gratuita mediante GitHub Pages

Puedes desplegar esta aplicación web de manera 100% gratuita y en pocos segundos en GitHub Pages:

1. Ve a la pestaña Settings de tu repositorio en GitHub.

2. En el menú de la izquierda, haz clic en Pages (dentro de "Code and automation").

3. En la sección "Build and deployment", selecciona como origen Deploy from a branch.

4. Selecciona la rama main (o master) y la carpeta / (root), luego haz clic en Save.

5. ¡Listo! En 1-2 minutos, GitHub te proporcionará una dirección web pública tipo https://TU_USUARIO.github.io/TU_REPOSITORIO/ para acceder a la aplicación desde cualquier dispositivo.
